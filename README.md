## ✅ What Changed from Stage 0

- Added **Edit Mode** (update title, description, priority, due date)
- Introduced **Status Control** (Pending, In Progress, Done)
- Synced **checkbox with status**
- Implemented **expand/collapse description**

- Improved **time handling (days, hours, minutes)**
- Added **overdue detection**
- Improved **UI styling and layout**

---

## 🎨 Design Decisions

- Used a **card layout** for clarity and focus
- Applied **color indicators** for priority levels
- Kept UI minimal and readable for usability
- Used **flexbox for responsiveness**
- Designed with **mobile-first approach**

---

## ⚠️ Known Limitations

- No persistent storage (data resets on refresh)
- Edit form does not trap focus (basic accessibility only)
- No animations implemented
- Time updates every 60 seconds (not real-time seconds)

---

## ♿ Accessibility Notes

- Used semantic HTML elements (`article`, `time`, `button`, `label`)
- Checkbox is fully accessible and keyboard navigable
- Added `aria-expanded` and `aria-controls` for collapse toggle
- Time updates use `aria-live="polite"`
- Buttons are keyboard focusable
- Layout works across screen sizes (320px – desktop)
